Git is a version Control System which means it's a tool which can be integrated with our system itself or our code editor to track any or all changed made to our code with proper timelines and also who made those changes can be tracked as well and to do so we have to sync git with our system.


GitHub is basically a platform for us to upload or track changes or store our codes and also we can see other people's codes as well and the first step to uploading our codes to GitHub is to make a repository on github which is also called "repo" and a repo is basically a folder for our code and every change we make in it will be visible to us.
Now if we want to work with our directory here in our vs code then we need to use the "clone" command in our terminal while we are present in the folder of our repository and this is how the command works-- "git clone " now our terminal is working in the GitDemo folder but to work in the repo folder which is Love1-demo we need to use the command "cd" which stands for "change directory" and directory means a folder. Also while typing commands, using "tab" can help in autocompletion.
Now there are two types of files in a directory--- 1. Visible files 2. Non visible sytem files, ofcourse the files of the 2nd category are not visible by default so to see all the files present we need to use the "ls -Force" command.
Also all the files being tracked by git or working in sync with git will have a ".git" file in them which is hidden and can be seen with the command i wrote above.

Now if we make a new file or make any changes in our already present files those changes will not automatically be updated on our GitHub which means our repo on github and our repo on VS code will be different and when the files are not in sync with our GitHub then VS code displays that through certain symbols in front of our files like in this image.

Here "U" means Untracked means a file is not being tracked by git and is made recently and then we also have "M" which indicates a file is modified and is not ready for commit. The state for being ready for commit is called "staged for commit" and the status of all files can be checked by the "git status" command.

Now to add our changes to our repo on our remote system we need to go through a 3-stage process which is-- add, commit and push. All 3 of these processes are commands and adding means adding git to our untracked files, commiting means noting the changes made kind of like taking a screenshot of the changes and then at last we finalise the changes and we push them on our remote repo through the "git push origin main"--now here origin is the name we gave to our original repo that we made and main is the name of the branch in that repo.

Now we have learned about cd command ls command and many more and we also learned how to make a directory in GitHub now we will learn how to add changes or basically making a new folder and adding it's changes directly to GitHub in a new repo and to do that first we have to get out of our current directory and to do that we have the "cd .." command it takes us out of the current directory.To create a new directory or a folder through our terminal we have "mkdir directory name" command, this makes a new folder in our current workspace and then to add git in this new directory we use the "git init directory name" command to add git to our new directory. Now that we have added this directory to git we want to add all of it's changes to a repo and for that we need to set an origin repo for it and to do that we have to create a new repo in GitHub and then we have to set that repo as our origin for our changes to be pushed on GitHub using the "git remote add origin " this sets that new repo as our origin repo for us to add our changes to. We can also check if our local repo has been set to our desired repo on github through the "git remote -v" command, this will tell us if we have set our origin to the repo that we wanted to. Now there can be several branches to one git repo for example several people want to work on the same directory but each has a different role so they all can have a branch to themselves on which they can work and add, commit and push the changes onto, now by default we get the "master branch" but the name everywhere has been changed to "main" instead of master so first we check our branch using "git branch" and then if the branch isn't main then we can change it's name to main using "git branch -M main" and then we can push the changes using "git push origin main" and now if i know that all the changes i make will be uploaded to the origin repo and main branch then we can set it as default for our current directory using "git push-u origin main" this tells git that we want to set "origin main" as our default for everything so we don't need to specify everytime.
